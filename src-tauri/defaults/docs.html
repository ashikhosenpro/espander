Espander Documentation

Use Espander to manage Espanso snippets visually, then keep them local, import from Google Sheets, or sync two-way with GitHub YAML files.

App Guide

  1. Create categories from Settings, then reorder them by dragging.
  2. Click New Snippet, enter a trigger, replacement, and category.
  3. Use the sidebar categories to filter snippets quickly.
  4. After saving snippets, Espander can generate YAML and reload Espanso.
Triggers normally start with a colon, for example :sig or :email.

Local Provider

  1. Go to Settings, choose Sync Provider: Local Only.
  2. Manage snippets inside Espander. Data stays in ~/.espander/database.
  3. Set the Espanso path and config directory if auto-detect does not find them.
  4. Use Export CSV/YAML for manual backup.

Google Sheets Provider

  1. Create a Google Sheet with columns: trigger, replace, category, description.
  2. Publish the sheet to the web as CSV from File, Share, Publish to web.
  3. Copy the generated CSV link.
  4. Paste it in Settings under Google Sheets connection, then Sync Now.
Google Sheets import is one-way. Use GitHub for two-way read/write sync.

GitHub Provider

  1. Create a public or private GitHub repository for snippets.
  2. Create a Fine-grained Personal Access Token from GitHub Settings.
  3. Repository access: choose Only select repositories, then select your snippets repository.
  4. Permissions: set Contents to Read and Write, Metadata to Read-only.
  5. Paste the repository URL and token in Espander, test, connect, then Sync Now.
Each YAML file inside snippets/ becomes a category. For example new.yml becomes the New category.

Import/Export

  • CSV export includes trigger, replacement, category, description, and tags.
  • YAML export keeps categories and snippets together in a portable backup.
  • YAML import also accepts Espanso-style files with a top-level matches list.
  • Missing categories are created automatically during import.

Troubleshooting

  • If snippets do not expand, verify Espanso is installed and running.
  • On macOS, enable Accessibility permission for the text expansion tool.
  • If GitHub sync fails, test the token and confirm Contents Read/Write permission.
  • If a new GitHub YAML file does not appear, click Sync Now and then Refresh if needed.
